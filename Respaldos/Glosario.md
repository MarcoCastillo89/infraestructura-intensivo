# Diccionario de Backups

## 1. Backup Full
Es un respaldo donde se guarda toda la información completa cada vez que se realiza.  
**Ejemplo:** Cada domingo se hace una copia completa de todos los archivos de la empresa.

## 2. Backup Incremental
Solo guarda los archivos que cambiaron desde el último respaldo realizado.  
**Ejemplo:** El martes únicamente se respaldan los documentos modificados el lunes.

## 3. Backup Diferencial
Guarda todos los cambios hechos desde el último respaldo completo.  
**Ejemplo:** El jueves se copian todos los archivos modificados desde el domingo.

## 4. Backup Sintético
Es un respaldo creado usando copias anteriores sin volver a copiar todos los datos.  
**Ejemplo:** El sistema une varios respaldos pequeños para formar uno completo.

## 5. Forever Incremental
Se hace un respaldo completo una sola vez y después solo se guardan cambios nuevos.  
**Ejemplo:** Una empresa evita hacer copias completas cada semana para ahorrar espacio.

## 6. Replicación
Consiste en copiar información automáticamente hacia otro lugar casi en tiempo real.  
**Ejemplo:** Un servidor secundario recibe los mismos datos del principal.

## 7. Archivado
Es guardar información antigua que ya no se usa mucho, pero que puede necesitarse después.  
**Ejemplo:** Una escuela guarda documentos de alumnos egresados de años anteriores.

## 8. RPO
Indica la cantidad máxima de información que se puede perder después de un problema.  
**Ejemplo:** Si el RPO es de 30 minutos, solo se aceptarían perder datos de media hora.

## 9. RTO
Es el tiempo máximo permitido para recuperar un sistema después de una falla.  
**Ejemplo:** La página web debe volver a funcionar en menos de una hora.

## 10. Regla 3-2-1-1-0
Método para proteger información usando varias copias en diferentes lugares.  
**Ejemplo:** Tener respaldos en computadora, disco externo y nube para mayor seguridad.

## 11. Snapshot
Es una imagen rápida del estado de un sistema en un momento exacto.  
**Ejemplo:** Antes de actualizar Windows se crea un snapshot por seguridad.

## 12. Restauración
Proceso de recuperar archivos o sistemas usando un respaldo.  
**Ejemplo:** Recuperar una tarea eliminada accidentalmente desde una copia de seguridad.

## 13. Disaster Recovery
Plan que ayuda a recuperar sistemas y datos después de un desastre.  
**Ejemplo:** Una empresa usa servidores alternos cuando falla la energía eléctrica.

## 14. NAS
Dispositivo conectado a la red para guardar y compartir archivos.  
**Ejemplo:** Los empleados almacenan documentos en un NAS de oficina.

## 15. SAN
Sistema de almacenamiento usado para manejar grandes cantidades de información rápidamente.  
**Ejemplo:** Un banco utiliza SAN para guardar datos de clientes.

## 16. Deduplicación
Proceso que evita guardar archivos repetidos para ahorrar espacio.  
**Ejemplo:** El sistema detecta fotos duplicadas y guarda solo una copia.

## 17. Compresión
Técnica que reduce el tamaño de los archivos para ocupar menos espacio.  
**Ejemplo:** Un respaldo de muchos documentos se comprime antes de guardarse.

## 18. Cifrado
Método para proteger información usando códigos de seguridad.  
**Ejemplo:** Los respaldos viajan protegidos para que nadie pueda leerlos.

## 19. Backup Offsite
Respaldo almacenado en un lugar diferente al sitio principal.  
**Ejemplo:** Una copia de seguridad se guarda en otra ciudad.

## 20. Backup Onsite
Respaldo almacenado en el mismo lugar donde están los equipos principales.  
**Ejemplo:** La empresa guarda copias en discos duros dentro de la oficina.

## 21. Nube Híbrida
Uso combinado de almacenamiento local y almacenamiento en internet.  
**Ejemplo:** Una empresa guarda archivos en servidores propios y también en la nube.

## 22. Redundancia
Consiste en tener recursos repetidos para evitar pérdidas por fallas.  
**Ejemplo:** Tener dos discos duros con la misma información.

## 23. RAID
Tecnología que une varios discos para mejorar seguridad o velocidad.  
**Ejemplo:** Si un disco falla, el sistema puede seguir funcionando con otro.

## 24. Alta Disponibilidad
Capacidad de un sistema para mantenerse funcionando casi todo el tiempo.  
**Ejemplo:** Una plataforma de streaming sigue activa aunque falle un servidor.

## 25. Failover
Cambio automático hacia otro sistema cuando ocurre una falla.  
**Ejemplo:** Si el servidor principal deja de funcionar, otro toma su lugar.

## 26. Data Center
Lugar donde se encuentran servidores y equipos importantes de tecnología.  
**Ejemplo:** Una empresa renta espacio en un centro de datos para sus servidores.

## 27. Bit Rot
Daño o corrupción de archivos debido al paso del tiempo.  
**Ejemplo:** Un video antiguo ya no puede abrirse correctamente.

## 28. Verificación de Backup
Proceso para comprobar que un respaldo funciona correctamente.  
**Ejemplo:** Se prueban archivos respaldados para confirmar que sí pueden recuperarse.

## 29. Retención
Tiempo durante el cual se conservan las copias de seguridad.  
**Ejemplo:** Los respaldos mensuales se guardan durante un año.

## 30. Recuperación Granular
Permite restaurar solo un archivo específico sin recuperar todo el respaldo.  
**Ejemplo:** Recuperar únicamente una foto eliminada sin restaurar toda la carpeta.
